Поток данных

  1. Frontend вызывает invoke('send_rpc', { method: 'prompt.submit', params })
  2. Rust пишет JSON-RPC в stdin Python процесса
  3. Python обрабатывает через tui_gateway.server.dispatch()
  4. Python пишет JSON-RPC ответы и события в stdout
  5. Rust читает stdout, парсит JSON, пушит события через app.emit('agent_event')
  6. Frontend слушает события через listen('agent_event')

Проблемы текущей архитектуры

❌ Сейчас

  • Сырой JSON из stdout — нет типизации событий
  • Нет разделения ответов и событий
  • Нет streaming — ответ приходит целиком
  • Нет обработки tool calls в реальном времени
  • Нет reconnect при падении процесса
  • Child process handle leaked

✅ С доработкой (из hermes-express)

  • Типизированные StreamEvent (Token, ToolProgress, Done, Error)
  • SSE streaming через Channel<StreamEvent>
  • Разделение команд и событий
  • Автоматический reconnect
  • Graceful shutdown child process
  • Health check + auth check

StreamEvent — типы событий

#[derive(Clone, Serialize)]
#[serde(tag = "type")]
pub enum StreamEvent {
    #[serde(rename = "token")]
    Token { content: String },
    
    #[serde(rename = "reasoning")]
    Reasoning { content: String },
    
    #[serde(rename = "tool_progress")]
    ToolProgress {
        tool: String,
        status: String,
        emoji: String,
        label: String,
        tool_call_id: Option,
    },
    
    #[serde(rename = "done")]
    Done,
    
    #[serde(rename = "error")]
    Error { content: String },
    
    #[serde(rename = "status")]
    Status { status: String },
}

Готовые компоненты

  • gateway.rs — GatewayClient с SSE streaming, rate limiter, health check
  • commands_gateway.rs — Tauri команды: stream_chat, health_check, gateway_auth_check
  • state.rs — AppState с GatewayClient, Database, ProcessManager
  • db.rs — SQLite для хранения сессий и сообщений
  • process.rs — управление процессами (spawn, kill, health)
